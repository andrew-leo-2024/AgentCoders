# Billing & DWI Lifecycle

## DWI (Delivered Work Item) Concept

AgentCoders uses **value-based pricing**. Customers pay per Delivered Work Item ‚Äî a complete, quality-verified deliverable. Internal costs (tokens, API calls, compute) are never exposed to customers.

## Complexity Estimation

Two-stage estimation by `ComplexityEstimator`:

### Stage 1: Keyword Heuristics

| Tier | Keywords |
|------|----------|
| XS | typo, rename, label, color, text change, bump version, update readme |
| S | add field, fix bug, validation, unit test, simple endpoint, config change |
| M | new endpoint, new component, refactor, migration, CRUD, form, api route |
| L | new service, authentication, authorization, database schema, workflow, pipeline |
| XL | architecture, microservice, real-time, event-driven, full redesign, cross-service |

**Score boosters:** file count, test count, description length.

### Stage 2: Claude Haiku Fallback

For ambiguous cases where keyword matching is inconclusive, calls Claude Haiku API for classification. Returns `{ tier, priceUsd, confidence, reason }`.

## Quality Gates

**All 6 criteria must pass for a DWI to be billable:**

| # | Gate | Redis Event | Field |
|---|------|-------------|-------|
| 1 | Work item exists | `dwi:work-item-created` | `workItemExists` |
| 2 | PR linked | `pr:linked` | `prLinked` |
| 3 | CI passes | `ci:completed` | `ciPassed` |
| 4 | PR approved | `pr:approved` | `prApproved` |
| 5 | PR merged | `pr:merged` | `prMerged` |
| 6 | Work item closed | `dwi:work-item-closed` | `workItemClosed` |

## Revert Protection

Post-merge quality enforcement:

- **Window:** 30 minutes after PR merge
- **Trigger:** CI failure on merged code within the window
- **Action:**
  1. `RevertManager` creates revert commit
  2. DWI status set to `reverted`
  3. `isBillable` set to `false`
  4. Notification sent via Telegram
- Stale merge tracking entries cleaned periodically

## Budget Enforcement

`BudgetEnforcer` monitors per-tenant and per-agent spending:

| Threshold | Event | Action |
|-----------|-------|--------|
| **80%** daily/monthly | `warning-80pct` ‚Üí `billing:budget-alert` | Warning notification |
| **100%** daily/monthly | `exceeded` ‚Üí `billing:budget-exceeded` | Agent forced to `idle` |
| Session cost limit | `session-limit` | Individual session terminated |

Spending aggregated from:
- DWI record prices (revenue side)
- Usage records from `UsageRecorder` (cost side)

## Internal Cost Tracking (AWU)

**AWU = Active Work Unit** ‚Äî 15-minute blocks of active agent time.

:::warning
AWU is for **internal cost accounting only**. Never exposed to customers.
:::

- Tracked by `UsageRecorder` via `psubscribe` to `{tenantId}:agent:*:progress`
- AWU tick timer runs every minute
- **Internal cost:** $0.10 per 15-minute AWU block
- Used for margin calculation in daily summary

## Invoice Generation

Value-focused invoices generated by `InvoiceGenerator`:

- Fetches completed, billable DWIs for a billing period
- Calculates savings vs human equivalent (base: **$150/hour**)
- **Customer sees:** work item titles, complexity tiers, DWI prices, total savings
- **Customer never sees:** tokens, AWUs, API calls, model costs
- Stores in `invoices` table and creates Stripe invoice
